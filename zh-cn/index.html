怎么使用这页

仓库主线其实很简单：先让 Claude 记住你、再让 Claude 自动做事、最后让 Claude 能像一个系统一样协作。

第一层：提高单次对话效率 Slash Commands、Memory、CLI、Checkpoints。先把 Claude 用顺手。
第二层：做自动化和集成 Skills、Hooks、MCP、Subagents。把“会用”升级成“会跑流程”。
第三层：做团队级能力封装 Advanced Features、Plugins。把个人经验打包成更稳定的工作方式。

按等级理解整套路线

Level 1 入门

目标：先把 Claude 用顺手。

  • Slash Commands：做快捷操作
  • Memory：让 Claude 记住项目规范
  • Checkpoints：安全回退和试错
  • CLI：掌握 `claude` 和 `claude -p`

Level 2 进阶

目标：让 Claude 开始自动工作。

  • Skills：把经验固化成能力
  • Hooks：在关键事件自动触发检查
  • MCP：接入 GitHub、数据库、文件系统
  • Subagents：把复杂任务拆给专门代理

Level 3 高阶

目标：从“会用”进化到“会设计系统”。

  • Planning / Auto / Tasks / Permissions
  • 多会话、后台任务、远程控制
  • 适合复杂改造、团队协作、长流程任务

最终能力

把零散功能打包成工作流与插件。

  • Plugins：一次安装一整套能力
  • 团队规范分发
  • 代码审查、文档生成、CI/CD 自动化

建议下一步

7 天学习节奏

如果你想快速建立全局感，可以按 7 天节奏推进。它比一次读完更有效，因为每天都有非常明确的练习目标。

Day 1
Slash Commands + Memory。让 Claude 从今天开始“按你的项目规范说话”。
Day 2
Checkpoints + CLI。学会安全试错，以及如何用 `claude -p` 进入脚本化使用。
Day 3
Skills。把代码审查、文档整理、重构建议变成可复用能力。
Day 4
Hooks。给写文件、执行命令等关键行为加质量守卫。
Day 5
MCP。让 Claude 能用 GitHub、数据库、文件系统等外部工具。
Day 6
Subagents。把复杂任务拆成调试、审查、测试、文档等多个角色。
Day 7
Advanced Features + Plugins。最后把能力拼成完整工作流。

15 分钟快速起步

如果你不想先学完整结构，先做这三件事，收益最大。

# 1. 安装第一个命令入口 mkdir -p .claude/commands cp 01-slash-commands/optimize.md .claude/commands/ # 2. 建立项目记忆 cp 02-memory/project-CLAUDE.md ./CLAUDE.md # 3. 在 Claude Code 里试用 /optimize
  • 你会马上理解命令入口如何提升效率。
  • 你会看到 `CLAUDE.md` 为什么是长期协作底座。
  • 这三步也是后面技能、hooks、子代理稳定工作的前提。
